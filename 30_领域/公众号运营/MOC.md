## 🎯 一句话定位（最终版 / 公众号简介一字不变）

```
大姚｜AI 自动化工作流实战

一个人用 AI + 自动化，搭出能顶一个团队的工作流。

实战 Hermes、Coze、n8n、飞书多维表、Obsidian、Codex：
把内容、运营、知识和客户服务，变成跑得起来的系统。
```

> 🎯 **师傅原话点睛**：「我卖的不是一个人取代团队的焦虑，而是让一个有业务的人，把重复工作交给 AI，把精力留给判断、创意和成交。」

## 👤 目标读者

**4 类核心读者**（从 dayao-writing-prompt 提炼）：
- **内容运营** — 重复选题排版发文的痛
- **小团队负责人** — 一个人如何覆盖 3+ 职能
- **想用 AI 提效的人** — 不知道从哪下手
- **B 端客户（最关键）** — 想把重复工作自动化，需要看见被验证的流程

**读者共同的诉求**：「**一个人怎么做到的**」，而不是「这个工具多么厉害」。

## 🧰 8 工具角色框架（避免滑回"AI 工具大杂烩"）

> ⚠️ 这是 v3 新增的——每个工具写之前，先想想它在这个框架里**扮演什么角色**。

| 工具 | 角色 | 在工作流里做什么 | 可写的"结果"举例 |
|------|------|---------------|---------|
| **Hermes / Coze** | AI 大脑与智能体编排 | 把模糊需求变成可执行任务 | 文章撰写、智能客服、内容编排 |
| **n8n** | 系统连接器与调度器 | 把应用、数据、消息、模型串起来 | 数据抓取、跨平台同步、定时任务 |
| **飞书多维表** | 轻量数据库 + 人工操作台 | 管素材、任务、状态、审核、回流 | 原子库、看板、审批流、回流统计 |
| **Obsidian** | 个人知识库 + 思考系统 | 把碎片经验沉淀成可调用资产 | 第二大脑、知识图谱、双向链接 |
| **WorkBuddy** | 具体岗位的 AI 执行工具 | 让某类日常工作直接提效 | 文档摘要、日报生成、PPT 念稿 |
| **Codex / Vibe Coding** | 定制工具开发能力 | 补上现有工具做不到的环节 | 查重小工具、自动质检、内容去重 |
| **中转站** | AI 基础设施 / 成本层 | 给工作流提供模型、统一接口和成本控制 | 多模型路由、按量计费、节省 80% 成本 |
| **公众号** | 公开实验场 | 展示"选题到发布"的真实自动化成果 | 写文章本身不再是难点 |

> 💡 **写作含义**：表里的"可写的结果"列 → 直接拿去做选题库。8 个工具不变成"工具测评合集"，而是变成"我有什么成果可以分享"。

**协同示例**（写文章时这样组合）：
- 选题采集：n8n（定时）→ 飞书多维表（沉淀）→ Hermes（评分）
- 内容生产：飞书（取料）→ Hermes（写稿）→ 飞书（审核）→ 公众号（发布）
- 数据回流：公众号 → 次幂数据 API → 飞书（统计）→ Obsidian（复盘）

## 🧱 内容主线五段式（替代之前的 5 种文章类型）

> 这是 v3 师傅提的关键升级 — **5 段式** 比"5 种文章类型"更可落地。

**5 段式** = 一篇文章 5 段（不可缺）：

```
1. 原来什么工作卡住了：谁在做、耗时多少、为什么痛
2. 我如何拆流程：哪些给 AI、哪些给自动化、哪些必须留给人
3. 工具怎么组合：Hermes / Coze / n8n / 飞书 各做什么
4. 跑出来的结果和坑：效果、成本、异常、人审节点
5. 读者能怎么拿走：模板、思路、教程、服务入口
```

**核心理念**：「**人工审核**」一定要持续出现。成熟的自动化不是全无人，而是机器吃掉重复劳动，人处理判断、例外和高风险决策。

## 📅 5 个固定栏目（每周轮转）

> v3 新增。师傅给的方案 — 这 5 个栏目**直接对外讲**，比"5 种文章类型"更客户友好。

| 栏目 | 核心问题 | 内容例子 |
|------|---------|---------|
| **一人作战室** | 我如何把自己的工作交给系统 | 从热点采集到公众号草稿，我的内容流怎么跑 |
| **工作流拆解** | 一个具体业务环节如何自动化 | n8n + 飞书自动把客户资料归档、提醒、生成摘要 |
| **真实项目复盘** | 给客户或自己做了什么 | 客户案例：~150 条素材如何变成一年选题库 |
| **新工具上生产线** | 新工具值不值得接进系统 | WorkBuddy / Codex 值不值得放进内容生产链 |
| **自动化赚钱实验** | 工作流如何成为产品或服务 | 中转站如何作为工作流的模型成本层，而非单卖 Token |

**5 栏目关系**（写作时挑）：
- 「一人作战室」+「工作流拆解」 = 主力轮转（每周至少 1 篇）
- 「真实项目复盘」+「新工具上生产线」 = 转化/留存（每 2 周 1 篇）
- 「自动化赚钱实验」 = 长期变现铺垫（每月 1 篇）

## 🚨 写作边界硬约束（师傅定的发文门槛）

> ⚠️ 这是 v3 最重要的一条 — 守住这条线，不会滑回"AI 工具大杂烩"。

### ❌ 不写（红线）
- "WorkBuddy 好不好用"
- "Codex 又更新了什么"
- "这个插件推荐收藏"

### ✅ 只写（绿线）
- "我发现了一个重复环节 → 用某工具搭了流程 → 中间哪里翻车 → 最终省掉什么 → 你怎么复用 / 我可否帮你搭"

### 三个例子

| 不写（错） | 写（对） |
|----------|--------|
| WorkBuddy 实测：这个工具到底怎么样 | 运营每天花 2 小时找资料，我把它拆成 WorkBuddy + 飞书工作流后，只保留最后 15 分钟审核 |
| n8n 如何连接飞书多维表 | 我用 n8n + 飞书，把 ~150 条客户资料洗成可日更一年的公众号原子库 |
| Codex 入门教程 | 现成工具总差最后一步：我让 Codex 给飞书内容库补了一个「自动查重+选题评分」小工具 |

## 🧠 我对这个领域的关键判断（v3 终版）

1. **8 工具归类是边界** — 写到飞书/Hermes/n8n 时，按这个角色框架讲，不写工具测评
2. **5 段式是骨架** — 不漏段，特别第 5 段"读者能怎么拿走"必须出现
3. **5 个栏目是对外语言** — 文章标题可以包含栏目名（如「一人作战室 #1」）
4. **"人工审核"持续出现** — 这是核心信任资产，不要为了"全自动"丢掉
5. **~150 条标杆项目是锚点** — 必须进 MOC 强凭证，否则跟"AI 工具号"无差异
6. **数据为王** — 飞书 13 原子库是唯一真实数据源，postflight 卡禁用词
