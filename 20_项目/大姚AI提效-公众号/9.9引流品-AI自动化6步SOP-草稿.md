## 一句话

## 这份 SOP 给谁

写给**月入 1-3 万的内容 / 运营 / 小团队负责人**——想从"我一个人干活"变成"我顶一个团队"。

3. **从"内容来一篇少一篇"到"每天 30 秒出 30 条选题"**——5 步动线把读者留下

## 🚦 STEP 2：选题（自动化跑得最彻底）

**核心问题**：v5 1548 字、阅读 100、分享 0、加微 0——**真因是选题 + 画像错配**。

### 选题的 3 个反向原则

| # | 反着来 | 原因 |
|---|--------|------|
| 1 | **先找对标，再写自己**——不是"我擅长什么写什么" | 写作有壁垒，先抄借鉴 |
| 2 | **看一个具体的人遇到具体问题，不看"AI 工具大全"** | 兴趣画像吸引围观，生存画像吸引客户 |
| 3 | **标题必须有数字或"具体场景"**——不写"AI 自动化未来已来" | 算法要可识别，读者要明确 |

### 3 工具协同选题公式（爆款）

```

```

**套用到你的方向**：X 工具 + Y 工具 + Z 工具 搭一个 [具体日常场景] 工作流。

## 🚦 STEP 4：内容框架（14 条规则 = 5 段式 · Clippings 级颗粒度）

> 💡 这一章是 SOP 的灵魂。每个段都给"师傅实测长度 + 3 个 hooks 例子 + 1 个师傅自己写的样本"。你抄模板即可干出 1500-1800 字文。
>
> ⚠️ **三条硬约束**（破了一条 = 不合格）：
> 1. ❌ 没有"具体的人 + 具体的场景 + 具体的问题 + 具体的数字"4 元素 → 标题改 3 次
> 2. ❌ 第 3 段（工具）只写方法不给命令 → 加上代码/配置
> 3. ❌ 第 4 段（结果和坑）写"成功"不写"翻车" → 必须有真实坑

### 4.1 五段式骨架（师傅实测长度）

| 段 | 名字 | **实测字数** | 作用 | 失败模式 |
|----|------|------------|------|---------|
| 1 | **卡** | 50-100 | 一句话钩子，制造"读下去" | 写成"导语"——不够尖锐 |
| 2 | **拆** | 300-500 | 把卡里的概念拆开 3-5 点 | 列了 7-8 点——读者读不完 |
| 3 | **工具** | 200-300 | 具体动作 / 命令 / 代码 / 步骤 | 只写"用 X 工具"——读者没法抄 |
| 4 | **结果和坑** | 200-400 | 自己实际做了的过程 + **3-5 个真实坑** | 只写成功——读者羡慕完就走 |
| 5 | **读者能拿走** | 100-200 | "你能拿走什么"——3 个清单 + 1 个动作 | 写成鸡汤——读者无感 |
| 附 | **边界** | 50-100 | 方案不能做什么 | 不写——读者对你的期待失控 |

> 📐 全部加起来 **900-1400 字**，远低于行业惯例 2000+。少而精胜过多而糙。

### 4.2 师傅实测 3 个 hooks 例子（卡的写法）

> ⚠️ "卡"那段决定打开率。下面 3 种结构，**任选 1 个用**：

#### Hook A: 数字 + 反差
```
v5 1548 字、阅读 100、分享 0、加微 0。
不是写挫——是**写错了**。
```

#### Hook B: 场景代入 + 问题
```
昨晚 11 点，小赵在 4 个公众号前崩溃——
老板让她"明天把视频号也接上"，她没有选题。
```

#### Hook C: 自己 + 金句
```
我剪完 18 篇、跑通 14 个对标、改完简介、搭好动线——**公众号还是 168 粉**。
不是平台薄情，是我错了 1 个变量。
```

> 🎯 师傅自己的 v6 全文就是 Hook A 风格——可以照搬。

### 4.3 拆：4 个最常见的拆法（任选 1）

### 4.5 结果和坑：3 条法则

> ⚠️ **不写"成功"写"翻车"**——这是 v3 定位的反差点（其他人都在晒成果，你在翻车 = 真实）。

| # | 法则 | 师傅实战 |
|---|------|------|
| 1 | 翻车必须"具体" | 不写"工具卡了"，写"次幂数据 check-balance 返回 404，根因：host 是错的，原写 `http://api.cimidata.com` 已失效，要改 `https://www.cimidata.com/`" |
| 2 | 给出"解决路径" | 不写"后来搞定了"，写"查 cimi-data-mcp 的 GitHub 源码，找到 `/api/token` 端点 + body 用 `{app_id, app_secret}` + token 用 query string 鉴权" |
| 3 | 标"教训类"标签 | 不写"AI 工具要选对"，写"v4 启动第一个坑：token 鉴权失败，因为没看官方 MCP 客户端源码自己揣" |

### 6.3 决策树：哪类砍 / 哪类复制（每周日用）

```
本周发了 5 篇
↓
用 09 库数据算"阅读 / 加 V / 完读率"
↓
决策树：

Q1: 阅读 ≥ 1500 + 加 V ≥ 5？
  YES → 拆 WHY：标题哪里好 / 卡 hook 用了哪个 / 工具给了什么命令
       → 把"师傅判断"写进 13 库（观点型原子）
       → 下周同一栏目出 +3 篇同结构
       ✅ 复制

  NO → Q2

Q2: 阅读 500-1500（有点起色但不够爆）
  → 看"卡 hook 用了哪个"：是 Hook A 数字反差型？
  → 是：换 Hook B 或 C 重写一次
  → 不是：换标题测一次（A/B test）
  ⚠️ 测试（不复制也不砍，看下周数据）

  NO → Q3

Q3: 阅读 < 500
  → 看师傅判断字段（如果填了）
  → 填了"踩坑"：把坑当反面案例（下一个内容讲"为什么不写 X"）
  → 没填：直接砍，下周不写同类

✅ 决策结果 = 本周 3 件小事：
  1. 复制 = 哪篇？扩成 3 篇
  2. 测试 = 哪篇？换 Hook 重发
  3. 砍 = 哪类？记进"避坑清单"
```
