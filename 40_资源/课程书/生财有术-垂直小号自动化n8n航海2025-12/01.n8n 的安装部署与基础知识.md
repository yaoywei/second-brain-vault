1.1 对比了解 N8N 部署方式

n8n 主要分为两大版本：官方云版 （n8n Cloud） 和 自托管版 （Self-Hosted）。

官方提供的云服务版本，是 n8n 最便捷的入门方式。你无需关心任何技术细节，只需注册账号即可开箱即用。

优点：

零技术门槛：无需任何服务器知识，注册即用。

持续运行：由 n8n 官方维护，保障 7x24 小时稳定在线运行。

自动更新：无需手动操作，系统会自动更新到最新版本。

免费试用：提供 14 天免费试用。

缺点：

价格较高：相较于自托管，长期使用的成本更高。

灵活性受限：无法进行深度定制或安装社区开发的非官方节点。

数据隐私：因为是官方云服务，无法保证完全的数据隐私

适用人群：

希望快速体验 n8n 核心功能的用户，或者纯粹不差钱的

1.1.2 自托管版 （Self-Hosted） 【免费】

所谓自托管，就是利用 n8n 官方免费提供的开源程序，将其安装在你自己的设备上。这个“设备”可以是你的个人电脑，也可以是你购买的云服务器。自托管版进一步可以分为两种主流方式：本地部署 和 服务器部署。

A. 本地部署/PC 部署【航海主推款】

将 n8n 直接安装在你自己的电脑（Windows、macOS 或 Linux）上。如果你想零成本地长期学习和使用 n8n，并且不要求工作流 24 小时在线，那么在你的个人电脑上进行本地部署是最佳选择

核心优势：

零成本： 完全免费，只要你的电脑能开机，就能运行 n8n。

数据安全： 所有数据和凭证都存储在你自己的电脑上，不经过任何第三方。

调试方便： 直接在本地创建和测试工作流，响应速度快。

核心劣势：

无法 7x24 小时运行： 你的电脑关机或休眠后，n8n 就会停止运行。这意味着依赖 Webhook 触发或定时执行的工作流将无法正常工作。

依赖本地网络： 无法从外网直接访问你的 n8n 实例（除非进行复杂的内网穿透设置）。

适用人群：

本次航海进行学习和测试的主推方式

B. 服务器部署 （Server Deployment）【进阶推荐版】

这种方式将 n8n 安装在一台你拥有的云服务器上（如阿里云、腾讯云、华为云或海外的 VPS）是功能最完整的自托管方式。这是“一次投入，长久受益”的方案。虽然前期有少量学习成本，但它能让你彻底拥有一个全天候运行工作流的场所。对于本次航海的大部分应用场景，服务器部署是最终的理想形态。

核心优势：

兼具可控性与可用性： 实现了 7x24 小时在线运行，同时你拥有对 n8n 实例和数据的完全控制权。

成本可控： 相比官方云版，自己购买一台入门级云服务器的成本要低得多。

性能灵活： 可以根据工作流的复杂度和执行频率，随时升级服务器配置。

可公开访问： 配置域名后，可以像访问普通网站一样，随时随地通过浏览器访问你的 n8n。

核心劣势：

需要一定的技术能力： 上手相对较难

需要自行维护： 包括环境配置、版本升级、安全防护、问题排查等都需要自己负责。

适用人群：

本次航海进阶方式，许多需要自动定时（每天、每周）执行的工作流需要此部署方式

1.3 n8n 本地部署 Windows 版（用 Docker）【航海主推款】

为什么Windows安装docker容易出现报错：

Docker 本质上是基于 Linux 系统的工具。Mac 电脑因为系统内核相近，安装起来很顺滑。但 Windows 系统要运行 Docker，就相当于要在一个讲中文的环境里强行运行一个全英文的部门。

因此，我们需要先在 Windows 里搭建一个“翻译层”（虚拟环境），这个过程对新手来说可能有点繁琐，涉及系统设置和命令输入，且听起来有点复杂，但别担心，我们只要按顺序把基础环境搭好，后面安装就非常顺畅了，接下来用详细的步骤带大家一步步完成部署。

在开始动手之前，请确保你满足以下两个硬性条件，否则后面的步骤无法进行。

第一个：网络环境（非常重要）

Docker 的安装包和后续需要的组件（镜像）都在海外服务器。如果网络不稳定，你会经常遇到下载进度条不动、连接超时或者直接报错的情况。为了节省你的时间，强烈建议准备一个稳定的科学上网环境。由于合规原因，此处不作工具推荐。如果你遇到网络困难，可以寻求航海志愿者、领队和线上运营团队的帮助。

第二个：确认电脑芯片类型

为什么要确认电脑芯片

前面我们知道，Windows 系统要运行 Docker，需要先在 Windows 里搭建一个“翻译层”（虚拟环境），也就是在windows系统下模拟适合docker运行的虚拟环境。最好的安装教程和问题解决方案都在官方文档里，我们来参考官方最权威的说明： https://docs.docker.com/desktop/setup/install/windows-install/

可以看到，有两种不同的后端模拟方式，分别是Hyper-V和WSL 2（WSL 2是目前推荐的后端，它需要特定的Windows功能来创建轻量级虚拟机）。

并且对不同的操作系统和芯片指令集对应的前置要求不同，这首先就需要明确自己的电脑用的是什么操作系统版本和芯片指令类型（90%以上都是x86\_64指令集）

最简单的查看方法是：在桌面的“此电脑”或“我的电脑”图标上点击鼠标右键，选择属性。

在弹出的"系统信息"选项卡里找系统类型。

这里的系统类型必须明确(后续在安装docker要选择正确的安装包)

AMD64 (x64)：适用设备：绝大多数Windows电脑

处理器品牌：Intel Core（酷睿）系列（i3/i5/i7/i9）、AMD Ryzen（锐龙）系列、以及其他x86-64架构的处理器

ARM64：适用设备：较新的ARM架构设备

典型设备：Microsoft Surface Pro X、搭载高通骁龙处理器的Windows笔记本、苹果M1/M2/M3芯片的Mac（通过虚拟机运行Windows）

绝大多数人的电脑显示的都是“基于 x64 的处理器”，如果你是这类，后面直接下默认包就行；

如果显示的是“ARM64”（比如 Surface Pro X 或 Mac 虚拟机），请务必记住这一点，后面要下 ARM 版的包。

1.3.1 部署前准备动作：搭建 WSL 2 环境

WSL 2 (Windows Subsystem for Linux)，是微软出的一个“套娃”系统，允许你在 Windows 里跑真正的 Linux 内核。我们需要安装这个叫 WSL 2 的组件，它是目前官方最推荐的方案，能让 Windows 完美兼容 Docker

因为Docker 必须要用 Linux 内核才能工作，没有 WSL 2，Docker 就只是一个空壳软件，根本动不起来。

接下来我们需要通过命令行来安装它。以下步骤均需要在一个管理员权限的终端中进行。

我们要完成三个小任务：安装子系统 -> 开启系统开关 -> 确认虚拟化开启。

```
bcdedit /enum
```

在一大堆出来的字里，找一行叫 hypervisorlaunchtype 的东西。 如果它显示的是 Auto，那就完美了，不需要动。 如果它显示 Off，请立刻执行下面这行命令并回车，把它强制打开，否则 Docker 绝对启动不了：

```
bcdedit /set hypervisorlaunchtype Auto
```

AMD-V

理解 Docker 的三个核心概念

数据卷 Volume：

是数据持久化存储的地方，n8n的工作流、凭证等信息都会保存在数据卷中，非常重要，不能删除，一旦删除所有需要持久化存储的内容就会丢失！

数据卷相当于我们在硬盘上划出的一块“保存区”，用于存放 n8n 的工作流、账号凭证等重要信息。

镜像 Image：

这是应用程序的只读模板，包含了运行 n8n 所需的所有代码、运行库和环境配置。它相当于软件的安装包，我们可以从 Docker Hub（官方仓库）将其下载到本地。

通过docker pull命令拉回具体镜像，这个具体的镜像就会显示在镜像标签卡里，可以删除，而且一般n8n升级的时候需要删除再拉回最新的镜像版本。镜像就像一个模板，比如word应用程序，可以根据它来点开多个应用界面，同时打开文件1、文件2等等

容器 Container：

这是镜像运行起来的动态实例。如果我们把镜像看作是静态的代码，容器就是正在运行中的程序。我们真正使用 n8n 时，就是与这个正在运行的容器进行交互。

通过docker run命令运行，指定某一个镜像，加上对应的数据卷、环境变量等，是一个具体的实例。就好比上面说用word打开多个文件，word就是镜像，具体的每一个文件就是容器。也可以删除，不影响数据持久化存储。

挂载数据卷 -v n8n\_data:/home/node/.n8n 这一步是将第一步创建的“安全存储区”与容器内部存放数据的目录绑定。这样，产生的所有数据都会直接写入那个安全的硬盘空间，确保存储安全。

2.3 工作流构成基础知识

工作流 （Workflow）、节点 （Node） 与执行 （Execution）

在 n8n 的世界里，所有复杂的自动化都是由三个核心要素构成的。理解它们的关系，是掌握 n8n 的基石。我们可以用一个餐厅的比喻来生动地解释：

工作流 （Workflow）: 如同餐厅的“菜谱”。

它是一整套为完成特定自动化任务而设计的指令集。例如，“自动处理客户订单”就是一个工作流。它定义了从接收订单到发送通知的完整流程和蓝图。

节点 （Node）: 如同菜谱中的“配料”与“步骤”。

节点是构成工作流的基本建筑模块，每个节点代表一个具体的操作或一个步骤。例如，一个节点可能是“从 Google Sheets 读取新订单”，另一个节点是“用 OpenAI 总结订单”，再一个节点是“发送 Gmail 邮件”。你通过将这些节点按顺序连接起来，来完成整个“菜谱”的制作。

执行 （Execution）: 如同“顾客点餐并上菜”的过程。

执行是指一个工作流被实际运行一次的过程。当满足触发条件时（例如，一位顾客点了一份“巧克力蛋糕”），n8n 就会按照工作流（菜谱）的设定，从头到尾执行一遍所有节点（准备配料、混合、烘焙、上菜）。每一次成功的运行，都对应一次“执行”记录。你可以通过查看执行记录，来了解工作流每次运行的具体情况，这对于调试和优化至关重要。

可以看到，每个节点内部是这样三块区域

左侧是从上一个节点传入的数据

中间的当前节点的设置

右侧是当前节点输出的数据

红色按钮是执行当前节点

最右侧小图钉形状的按钮是固定当前节点的输出数据（这样有些付费的节点你就可以调用一次然后固定住再做后面测试，否则每次运行都要花钱）

触发器（Trigger）、节点（Node）、凭证（Credential）

上面的比喻帮助我们理解了宏观概念。现在，让我们聚焦于你亲手搭建工作流时必须面对的三个最关键的组成部分：触发器 （Trigger）、节点 （Node） 和 凭证 （Credential）。

这三者，才是在 n8n 画布上搭建自动化的“动手三件套”。

3.

触发器 （Trigger）

如果说工作流是一个“菜谱”，那么触发器就是决定“何时开始做这道菜”的指令。

每个工作流都必须有一个（且通常只有一个）起点，这个起点就是触发器节点。它负责“监听”某个特定事件，一旦事件发生，它就会被激活，并启动整个工作流的执行。

触发器主要分为三类：

手动触发 （Trigger Manually）:

工作方式： 最简单的触发器。工作流不会自动运行，必须由你在 n8n 界面上手动点击“执行”按钮。

适用场景： 测试和调试。 在你构建工作流的绝大部分时间里，都会用它来测试每一步是否按预期工作。

定时触发 （Schedule）:

工作方式： 按照你设定的时间规则自动运行。

示例： “每天早上 9 点运行一次”、“每 5 分钟运行一次”。

适用场景： 定时的数据同步、每日报告、定时清理任务等。

事件触发 （Event-based）:

节点示例： Webhook， Gmail (On message)， Google Sheets (On row added)

工作方式： 7x24 小时监听一个外部服务的特定事件。

示例： “当有人提交了表单时（Webhook）”、“当收到一封特定邮件时”、“当表格新增了一行数据时”。

4.

节点 （Node）

如果触发器是“启动”，那么普通节点就是“执行”。

在 n8n 中，除了那个触发器，其他所有的“步骤”都统称为节点。每个节点都封装了一个特定的功能，它接收上一个节点传来的数据，对其进行处理，然后再传给下一个节点。

n8n 提供了数百个节点，我们可以粗略地将其分为几类，这里不需要你完全了解，大概有概念就可以：

服务节点 （Service Nodes）:

示例： OpenAI， Google Sheets， Slack， MySQL， HTTP Request

功能： 与外部应用或服务进行交互（读取数据、写入数据、调用 API）。这是 n8n 最核心的价值所在。

逻辑节点 （Logic Nodes）:

示例： IF （条件判断）， Switch （多重判断）， Merge （合并数据）

功能： 控制工作流的走向。例如，“如果邮件包含‘紧急’，则发送到 Slack；否则存入表格”。

数据处理节点 （Data Processing Nodes）:

示例： Set （设置新数据）， Edit Fields （编辑字段）， Code （运行代码）

功能： 对数据进行“精加工”。例如，格式化日期、提取文本中的特定信息、合并两个数据字段等。

5.

凭证 （Credential）

一句话解释凭证：访问外部服务的“钥匙”或“通行证”。

当你使用一个“服务节点”（如 Google Sheets 或 Gmail）时，n8n 怎么知道它应该去操作你的谷歌账户，而不是别人的？n8n 怎么调用那些付费的 AI 服务，并且告诉 AI 平台扣费你的账户？这就需要凭证。

凭证是连接 n8n 和外部世界的安全桥梁。你只需要设置一次，就可以在所有工作流中重复使用。

•

工作方式：

a.

你需要在 n8n 的“Credentials”区域，或者节点内部，安全地添加一次你的账户信息（例如，通过 OAuth2.0 授权 n8n 访问你的谷歌账户，或者填入 OpenAI 的 API Key）。

b.

n8n 会将这个“钥匙”加密并安全地存储起来。

c.

当你在工作流中使用 Google Sheets 节点时，你只需从下拉菜单中选择你之前保存好的那个凭证（“钥匙”）。

d.

n8n 会在执行时自动帮你“开门”，而你的工作流蓝图本身不包含任何敏感密码。
