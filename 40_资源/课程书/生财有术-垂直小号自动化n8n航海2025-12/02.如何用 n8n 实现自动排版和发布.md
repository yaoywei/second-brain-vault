## 垂直小号自动化（n8n 工作流）丨实战手册丨2025年12月航海

## 02.如何用 n8n 实现自动排版和发布

上一章给大家详细的介绍了，如何用 n8n 搭建一个 AI 生成内容的工作流，但在内容生成后，我们会面临一个新问题，那就是内容是纯文字，没有排版，不是很好看，所以，今天将带大家打造一个 n8n 公众号 AI 排版工作流，做好以后，它既可以单独用，也可以接入上一节的 AI 写作工作流，实现全自动“生成+排版+发布”

⚠️注意：请在完成必修篇内容后，再来学习这一节的内容

首先，我们需要添加一个子工作流触发节点，在 n8n 画布空白处点击 + 号，我们在右边添加节点处搜索 “sub”找到 Execute Sub-workflow

这个触发节点是什么意思呢？

这个触发节点专门用于设计子工作流的，什么是子工作流？就是可以被其他工作流调用的工作流，有的时候，有些功能，可能很多工作流都会用到，为了避免重复的造轮子，我们可以提前封装公用的功能变成子工作流，后续直接引用就行了。

而子工作流必须使用这个触发节点作为开始，以接收上级工作流的参数。上级工作流则通过 Execute A Sub workflow 调用子工作流。这个我们在最后还会讲到。

AI Agent：这是“AI 大脑”，用来设置提示词（Prompt），告诉 AI 要干什么

OpenRouter：这是“AI 心脏”，用来连接具体的大模型。它是一个模型聚合器，用一个 Key 就能调用 GPT、Llama 等各种模型，非常方便，而且有免费额度。

如何配置？

双击 AI Agent 节点。

User Prompt Source：选择 Define below （在下方定义）。

点击 “Add Option”，选择 System Prompt。

System Prompt （系统提示词）：具体见下面的附件

User Prompt （用户提示词）：

这里是给 AI 下达“具体任务”。我们需要把“内容”和“模板”两个变量传给它。

填入以下内容：

```
请帮我排版这段内容：
{{ $('When Executed by Another Workflow').item.json.content }}

请严格参考这个HTML样式：
{{ $json.html }}
```

注意： 上面的 $('\[节点名\]') 是 n8n 的“表达式语法”，用于精确引用前面节点的数据。请确保 When Executed by Another Workflow 是你第一个节点的实际名称。[AI排版提示词.txt](https://sphere-sh.oss-cn-shanghai.aliyuncs.com/private/docx/drive/media/CdgLbbF2EokAObxe8KzcnoKVnr1?Expires=1784292641&OSSAccessKeyId=LTAI5tDANFnEjK2qWnHwZKzr&Signature=B%2BHptlPA%2F%2FE4Wz0QwbI6pI5cmLI%3D)

进入大模型的节点，首先，我们需要设置一下，用户提示词来源选项为“Define below”，接着我们需要添加一个系统提示词选项，用于我们设置系统提示词

然后我们分别往这两个地方，填入我们的用户提示词和系统提示词，系统提示词有点长，大家记得提示词填入之后，需要把我们需要排版的内容和参考 html 参数替换进来

到这里所有节点就都搞定了，然后我们整体执行一下，成功生成排版代码

我们遇到了一个新问题： AI 的输出可能不稳定，有时会夹带“好的，这是您的代码：”之类的废话，导致 HTML 代码不干净，导致不好用

解决方法： 需要加个格式化输出的节点，强制 AI 按照我们规定的“格式”输出。

如何操作？

第一步：打开开关

回到 AI Agent 节点，打开 "Require Specific Output Format" 开关。

第二步：添加解析器节点

打开开关后，AI Agent 节点底部会出现一个新的 + 号插槽（用于连接 Tool/Output Parser）。

在画布任意位置点 +，搜索并添加 Structured Output Parser 节点。

第三步：连接节点

拖动 Structured Output Parser 节点的输出端（右侧灰点），连接到 AI Agent 节点底部的新插槽上。

第四步：配置解析器

双击 Structured Output Parser 节点打开配置。

Schema Type：选择 Generate From JSON Example。

JSON Example：在下方的框中，填入你希望 AI 输出的“格式示例”。这等于在告诉 AI：“你必须，且只能给我一个叫做 html 的字段”。

填入以下代码：

JSON

```
{
  "html": "生成的html代码"
}
```

检查结果：

全部配置好后，再次点击 AI Agent 节点上的“执行步骤”。

此时查看 AI Agent 节点的主输出（绿色分支），你会看到数据已经变得非常干净，被包裹在了一个 output 对象中，后续直接使用 html 参数就是完整的代码了

3.接入自动发布工作流

当你看到这里的时候，恭喜你，你的“AI 排版子工作流”已经完工了，记得点击右上角的开关保存它

当你完成上一节内容二创和自动发布的工作流后，那么你可以在之前的工作流中引入这个排版工作流，实现自动生成并排版，然后发布到公众号的功能

现在，我们回到上一节的“内容自动生成和发布”工作流中，把这个排版插件“插”进去

打开你的“上一个工作流”（即 AI 内容二创的工作流）。

找到“拆分media\_id”的那个节点

点击 AI Agent 节点之后的 + 号。

在弹出的搜索框中，搜索 Execute A Sub Workflow（执行子工作流），并添加它。

注意： 添加后，这个节点在画布上的默认名称可能是 Execute Workflow

双击打开你刚刚添加的 Execute Workflow 节点。

Workflow （工作流）： 在下拉框中，找到并选择你创建的“AI 排版子工作流”（你起什么名字就选哪个，把前面大模型生成的内容传入）

Parameters （参数）：

content：这里我们要把“主工作流”AI 生成的文章内容传递给“子工作流”。

点击 f(x) 添加表达式，填入：{{ $('json字段拆分').item.json.content }}

Markdown节点的作用是将markdown转换为html，但是咱们的子工作流输出的结果就是html，所以这个节点不需要了，点击原本工作流中“Markdown”节点上方的删除按钮

这是最后一步。找到“配置上传微信公众号草稿箱的参数”节点，即Markdown后面那个节点

双击打开它。

找到用于提交“文章内容”的字段 article。

删掉原来的变量，填入以下这行代码：

```
{{ $json.output.html }}
```

节点修改之后，需要重新运行检查一遍有没有其他遗漏问题。

再到公众号后台的草稿箱中看一下，一篇排版好的文章就出来了

注意： AI 排版不是 100%完美的，选定的参考代码还有优化空间，且 AI 不是那么稳定。如果发现瑕疵，你可能需要回到“AI 排版子工作流”中，去优化 Set 节点里的“默认 HTML 模板”，或者在 AI Agent 节点中优化你的 System Prompt（系统提示词），多尝试几次，找到最稳定的排版样式。

4\. 进阶篇：保持垂直小号个人风格一致性

在前面的内容中，我们已经用 n8n 搭建了选题库，实现了内容的二创、排版以及自动发布，考虑到我们的公众号是往垂直化、IP 化和日更的方向去发展，所以生成的内容保持风格的一致性是很有必要的

在进阶篇里面，我们会给大家拓展，用 n8n + 大模型 把“公众号写作风格”拆解成可复用的固定模板：先学对标账号的风格，再把素材丢给 AI 按模板生成；最终连 HTML 排版也自动化，保证垂直小号/矩阵号输出风格高度一致。

4.1 如何用 AI 学习并生成个人风格

准备 5-10 篇你认为同一风格很好的参考素材（比如 5 篇心理学的爆款文章），建立对标风格库

针对不同体裁和文章类型整理成不同的风格素材库，用于后续 AI 学习，形成不同的风格提示词

可以从第二节中的选题库里面找你喜欢的或者是符合你个人风格的素材

这一步直接用 AI 大模型即可，推荐 geimini 2.5 pro

将对标风格喂给 AI 让 AI 学习其中的优势，并整理其风格，便于 AI 后续学习

```
整理这些文章的风格，包括其用词、逻辑、行文机构等方面，以便于我后期可以复刻其风格生成新的文章内容
```

一个好的提示词，是有角色设定+风格限定+结构限制+参考案例，这样生成的内容效果能最大化 AI 的能力，当然 AI 尽可能要选用上下文能力和写作能力比较强的（推荐 Cluade 4/Gemini 2.5 pro）

上一步的风格总结+Few Shot+结构限制形成一个完整版本的提示词

```
# 播客《XXXX》稿件创作指令 (V6)

### 1. 角色扮演 (Persona)

*   **核心定位：** 一位在顶级律所工作的资深女律师。她思维敏捷，逻辑清晰，对热点事件有自己独到的法律见解。她不是高高在上的说教者，而是听众身边那个既专业又有趣的“法律界闺蜜”。

    *   **自信从容：** 说话不疾不徐，自带一种专业人士的掌控感。开场白通常是“哈喽大家好，欢迎来到《XXXX》，我是X律师。我们不只聊法律，也聊聊法律背后的人和事。”

你的任务，就是完美地代入“X律师”这个角色，将下方我提供的核心资料，转化成一篇充满她个人风格的、专业与趣味并存的播客稿。

### 3. 稿件核心要求 (Output Blueprint)

> 什么叫做重大损害，人类的语言从来都具有模糊性。
> 在道德伦理学上，关于一个事情是不是正义，一直有道义论和功利论的正义，
> 前者讲对错，后者讲利弊。
> 为什么我们不应该作弊？道义论认为作弊本身就是错误的，
> 但是功利论却认为作弊被抓的后果很严重啊。估计我们大部分人都是功利论的拥护者。
> 当然，在哲学上，这些问题千百年来争论不休啊。
> 刑法是一种最严厉的制裁措施，所以，在入赘问题上，
> 还是应该监采道义论和功利论。既要在行为上是错误的，
> 要在结果上是错误的，才能发动刑法权。
> 因此，只要行为或结果任何一项并无不当，那就可以排除犯罪。
> 如果防卫行为正当，但是结果不当，
> 或者防卫行为不当但结果正当，那都不属于防卫过当，
> 而是正当防卫。前者，你比如说一个女孩子在地铁遭遇险阻，
> 手拿书砸男方的头，这行为正不正当？
> 正当的不得了，但是男方心脏病突发而死，那这个结果可能不正当啊。
> 但是他行为正当，结果不当，那依然是正当方面。
> 那后者，比如说一个女生遭遇咸猪手，用膝盖顶男方下身啊，
> 导致男方轻伤，即便你认为他行为不当，但结果是正当的，
> 所以这都不能够以犯罪论处。从功利论的角度，
> 张三把人捅死了，结果可能不当，但是从道义论的角度，
> 他的行为合情合理。人之残疾，行为正当，因此不属于防卫过当，
> 而是正当防卫。正当防卫的本质是对民众的授权，
> 鼓励民众向不法侵害作斗争。法无需向不法让步，
> 绝对不能对于防卫人过于苛求，寒了见义勇为者的心。

> 更重要是维护自己的道德存在和人格。

> 是法港所产生的最美丽、最振奋人心的正眼。
> 定罪良刑，绝不能破坏民众有关健全法感的朴素思维。
> 更为重要的是，我们的人生应该讲对错还是讲利弊呢？
> 虽然柏拉图试图调和道义论和功利论的争论，
> 他认为正义就是那种最好的东西啊，他行为本身是好的，而且一定也会带来好的结果，
> 只是这种好的结果可能是在肉眼不可见的洞穴之外，
```
