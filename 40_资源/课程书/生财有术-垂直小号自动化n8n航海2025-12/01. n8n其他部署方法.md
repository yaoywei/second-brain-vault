核心配置项

Container name（容器名称）: n8n

给容器起的名字，方便管理

可以自定义，如"n8n-prod"、"n8n-test"

留空会自动生成随机名称

Ports（端口映射）

Host port: 5678（宿主机端口）

Container port::5678/tcp（容器端口）

访问地址：配置完成后通过 http://localhost:5678 访问 N8N 界面

```
5678
:5678/tcp
http://localhost:5678
```

端口映射原理：

容器内 N8N 运行在 5678 端口

允许从浏览器访问容器内的服务

如果 5678 端口被占用：

改为其他端口，如 8080

访问地址变为 http://localhost:8080

Volumes（数据卷挂载） ⭐ 最重要配置

Host path: D:\\project\\n8n（Windows 本地目录）

这是你 Windows 电脑上的实际文件夹

用于存储 N8N 的数据（工作流、凭证、文件等）

Container path: /home/node/.n8n/（容器内路径）

N8N 在容器内的数据存储位置

这是固定路径，不要修改

上面卡住或者无法下载必看：

因为 npm 源地址默认是海外服务器，受众所周知的原因，速度非常慢，甚至无法下载，这可以有两种解决方案。

💡

一、给命令行或者终端设置代理

如果你是一个追求高雅生活品质的人，那么你可以选择这个办法，它的做法非常不简单，

首先你要找到你用的【配置上网环境】软件的代理端口

如果你不知道这些！！！推荐你使用方法二。

知道端口之后，打开你的命令行输入以下两条命令：

npm config set proxy http://127.0.0.1:7890

npm config set https-proxy http://127.0.0.1:7890

注意：这里的 7890 要换成自己【配置上网环境】的端口

注意：这里的 7890 要换成自己【配置上网环境】的端口

注意：这里的 7890 要换成自己【配置上网环境】的端口

取消终端代理命令（你安装好 n8n 之后可以考虑按需取消代理，这个命令可以不执行，不影响安装）：

npm config delete proxy

npm config delete https-proxy

二、用镜像网站

直接无情输入：

npm config set registry https://registry.npmmirror.com

这个是用国内的镜像网站，有更新慢，兼容差的缺点，唯一好处是就算没有【配置上网环境】，下载也快

恢复海外版命令（你安装好 n8n 之后可以考虑按需恢复，这个命令可以不执行，不影响安装）：

npm config set registry https://registry.npmjs.org/

注意： 这里的命令输入，正常就是没有反应的，正常下一步就行，这里只是切换了下载源，你还要再次输入：npm i -g n8n

```
npm config set proxy http://127.0.0.1:7890     
npm config set https-proxy http://127.0.0.1:7890 
npm config delete proxy
npm config delete https-proxy
npm config set registry https://registry.npmmirror.com
npm config set registry https://registry.npmjs.org/
npm i -g n8n
```

命令行输入：n8n，然后等待跑数据，直到出现如下页面：

红框中有个 ip 地址，这个就是打开 n8n 页面，把它复制到浏览器就可以打开了，也可以按键盘的【o】自动打开。

以后每次要使用 n8n 网页，都要输入这个命令。
